# Installation

This guide covers the hardware-side install and the first data sync. The
Pip-Boy app cannot fetch weather by itself; it reads the cached
`USER/WEATHER.JSON` file written by the companion.

## Prerequisites

- The Wand Company Pip-Boy 3000 with SD card file access.
- Python 3 on the computer or phone that will run the companion.
- Internet access on the companion device during sync.
- The files in this repository.

Back up the entire SD card before changing it. The install is small, but the
backup protects the original firmware files and any local data already on the
card.

## SD Card File Map

Copy these files from the repository to the same relative locations on the SD
card:

| Repository path | SD card path | Purpose |
| --- | --- | --- |
| `pipboy/APPS/WEATHER.JS` | `APPS/WEATHER.JS` | On-device app code. |
| `pipboy/APPINFO/WEATHER.info` | `APPINFO/WEATHER.info` | App metadata and file registration. |
| `pipboy/APPINFO/WEATHER.IMG` | `APPINFO/WEATHER.IMG` | Holotape menu icon. |
| generated `WEATHER.JSON` | `USER/WEATHER.JSON` | Cached weather payload. |

The `USER/WEATHER.JSON` file is generated by the companion. Do not copy
`sample/WEATHER.JSON` for normal use unless you only want demo data.

## First Install

1. Connect the Pip-Boy over USB-C and open the web file manager, or remove the
   microSD card and mount it on your computer.
2. Back up the card.
3. Run the companion GUI:

   ```bash
   python companion/pipboy_weather_gui.py
   ```

4. Add or adjust locations and choose `F` or `C`.
5. Press `INSTALL / UPDATE DEVICE`, then select the SD card root. Examples:

   ```text
   Windows: E:\
   macOS:   /Volumes/PIPBOY
   Linux:   /media/<user>/PIPBOY
   ```

   This copies `WEATHER.JS`, `WEATHER.info`, and `WEATHER.IMG`, fetches
   Open-Meteo weather plus NOAA SWPC space weather, and writes
   `USER/WEATHER.JSON`.
6. Confirm that `<SD>/APPS/WEATHER.JS`, `<SD>/APPINFO/WEATHER.info`,
   `<SD>/APPINFO/WEATHER.IMG`, and `<SD>/USER/WEATHER.JSON` exist.
7. Eject the card or disconnect the USB session cleanly.
8. Reboot the Pip-Boy.
9. Open `ITEMS > MISC > Weather`.

## Manual Data Copy

If you do not set an SD path in the companion, it writes:

```text
companion/WEATHER.JSON
```

Copy that file manually to:

```text
<SD>/USER/WEATHER.JSON
```

This is useful if the companion runs on a phone or if you prefer to transfer
the cache separately.

## Updating an Existing Install

For app updates, replace:

```text
APPS/WEATHER.JS
APPINFO/WEATHER.info
APPINFO/WEATHER.IMG
```

Then reboot the Pip-Boy. Existing `USER/WEATHER.JSON` data can stay in place,
but syncing again is recommended when the data format changes or when the cache
is stale.

## Uninstalling

Remove these files from the SD card:

```text
APPS/WEATHER.JS
APPINFO/WEATHER.info
APPINFO/WEATHER.IMG
USER/WEATHER.JSON
```

Reboot the Pip-Boy after removal so the app list refreshes.

## Verifying the Install

A working install has all of these behaviors:

- `RobCo Weather` appears in `ITEMS > MISC`.
- Opening the app shows the first saved location.
- The scroll wheel changes location.
- The thumbwheel changes between `ATMOS`, `5-DAY`, and `SOLAR`.
- The footer shows the last sync timestamp.

If the app opens but reports `NO WEATHER DATA`, the app files are installed but
`USER/WEATHER.JSON` is missing or unreadable.
